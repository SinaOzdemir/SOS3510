Chapter 1 From untidy text to tidy tabular data

1.1 What is a tidy tabular data

Textual data in its natural form is called unstructured data. This is because it is not in a matrix format and does not have associated numeric values. The first step in natural language processing is to transform this unstructured data into structured tabular data. There are three methods to do this: bag-of-words approach, n gram approach and word embedding. We will not get into word embeddings in this lecture for three reasons. First, other text representations word embeddings, and deep learning makes heavy use of linear algebra and tensor algebra which most of us are not really familiar. Secondly, in most cases, word embeddings is mainly used with neural network models. These models require a lot of computational power which most of us don’t have. Lastly, neural network models require a lot of observations (usually more than 10.000). In most application of automated text analysis in social sciences, such a dataset does not exist. So we will mainly cover bag-of-words and n-gram approaches.

1.1.1 Bag-of-words approach

Bag-of-words approach creates a matrix out of the text where each row represents the unit of analysis (eg. sentence, paragraph, document) and each column represents words in the corpus1. In bag of words approach, documents are broken into tokens (eg. words) to create the dataset. The advantage of this approach is that it is usually quite fast and not so computationally demanding. This, however, comes at a cost. Bag-of-words approach cannot account for the order of words, hence the name. The metaphor is that we break the text into tokens and stuff them into a bag. There are three common ways of creating tabular datasets with bag-of-words approach. These are one-hot encoding, document-feature matrices and term-frequency inverse-document-frequency.

1.1.1.1 One-hot encoding method

This method represent the text as a matrix of binary values where rows are unit of analysis, columns are words and cells take a value of 1 or 0 indicating that a word is present in the unit of analysis. So now the sentence turned into matrix of 1x9 where each cell indicating that the word is present in a sentence. There are some advantages and disadvantages to this method. The key advantage is it is a simple representation so it can be done quickly and easily. Also binary representation of the text allows using some model such as Logic Regression. However, the simplicity is also a disadvantage. This representation does not tell us how many times a word is used in a document for example. Thus we lose a lot of information about the text when we use this method.

1.1.1.2 Document feature matrix

Another possible way of representing text with bag-of-words approach is using document feature matrix. The main difference of document feature matrix is that it can encode how many times a word used in a document. It can be easily created by using quanteda package as well.

  tokens() %>% 
  dfm() %>% 
  convert(to = "data.frame")

1.1.1.3 Term-frequency inverse-document-frequency (tf-idf)

The last method in bag-of-words approach is tf-idf. Building on document feature matrix, this method applies a weight to cells to alter their importance of some words. One commonly applied weight is to divide each word by how frequently they occur across documents so that rarer words gain more importance. This method can be very useful if what you are looking for may not be said frequently in a document. Again, we can easily obtain this with the quanteda package.

eu_speech_sample_tfidf<- corpus(eu_speech_sample,
                                docid_field = "doc_id",
                                text_field = "texts") %>% 
  tokens() %>% 
  dfm() %>% 
  dfm_tfidf() %>% 
  convert(to = "data.frame")

1.2 N-grams:

N-grams are the early attempt at preserving the word order in numeric representation of texts. Instead of using individual words as tokens, n grams combine multiple words to create tokens. It is called bi-gram if two consecutive words ar used as tokens, tri-gram if three words are used and so on. Lets see an example:

text<- "The quick brown fox jumps over the lazy dog"

text_tokens<- text %>%
  corpus() %>%
  tokens() %>%
  dfm() %>%
  convert(to = "data.frame")

head(text_tokens)
##   doc_id the quick brown fox jumps over lazy dog
## 1  text1   2     1     1   1     1    1    1   1
text_bigrams<- text %>%
  corpus() %>%
  tokens() %>% 
  tokens_ngrams(n = 2,skip = 0,concatenator = "_") %>% 
  dfm() %>% 
  convert(to = "data.frame")

head(text_bigrams)
