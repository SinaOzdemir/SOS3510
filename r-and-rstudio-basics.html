A quick introduction to automated text analysis

Chapter 3 R and Rstudio basics

Unlike STATA and SPSS, R is a programming language that is designed to do statistical operation. In that sense, R is a close relative of python. So when you do your analysis in R, you are actually writing statistical software! Advantage of R over STATA and SPSS is that you don’t have to wait for someone else to make a method available for you, you can simply create your own implementation. Downside of R, as we use it, it does not have point and click interface like STATA and SPSS. However, there are add-ons that provides such an interface.

We will start with very basics of R and Rstudio. First we will explore the interface and how to change it, because whats the point of using a programming language if you cannot add your own flare. Then we will move on to discuss basics of object oriented programming which is the corner stone of R. Once you are familiar with object oriented programming, we will move on to more practical aspects of R, namely scripting and sourcing. Then it will be time to learn some good practices when using R. Finally we will conclude with data operations such as data manipulation, indexing and sub-setting.

3.1 Interface & basics

When you start Rstudio first time, you will be greeted by three panels line in figure 3.1. The panel on the left is the console. It keeps track of operations that you make. Top right panel is called variable explorer or environment. This is the panel where you see your object such as data sets, functions etc. Bottom right panel is file explorer. This is where you will see help documents, plots and files. Take a look at 3.2 to see an example

Figure 3.1: Default Rstudio panels

Figure 3.2: How default Rstudio panels work

Perhaps one of the most common beginner mistake, and probably the most painful one, is to write your commands and functions directly into the console. this is problematic for two reasons. First, if you don’t finish your analysis in one sitting, all your work will be gone the moment when you shut down R (very much like STATA). More importantly, you won’t be able to experiment and replicate with different approaches to a problem without a script. So do not run anything directly on the console if you will need it again!. That being said, you can still use R console as a calculator. For example if you quickly need to see what \(3568794631298 * 321657165\) is you can simply type this in the console as 3568794631298 * 321657165 (it is 1.1479284^{21} if you are curious).

So what is a script and how do we get it? A script is essentially a text file where every line is a function. You can think of it as the word document for R (or .do files in STATA). It keeps what you write and helps you pass your operation to R to interpret and spit out results. To create a script, go to the tool bar on the top, click File> New File > R script or press CTRL+ALT+N or CMD+ALT+N (on Mac) as shown in the figure 3.3

Figure 3.3: Creating a new script

This will create a new panel on your interface. The new panel is the script panel where you can write your functions and analysis code. To execute each line individually, click on the line you would like to run and press CTRL+Enter on windows and CMD+Enter on Mac. This will pass the line you have written in the script to R to interpret and execute. You will get the results of an executed script line on your console as shown in figure 3.4

Figure 3.4: Using scripts

Now you can work on your analysis, and once you save your script, you can pick up where you left off later on. This brings us another important concept with R, that it working directory. The base R likes having all files in one folder. While we will discuss how to use relative working directory later on, it is important to know the concept of working directory first. When you start R and R studio, it will automatically pick your documents file as the base working directory. You can check where this is by running the following command on your console

getwd()
## [1] "C:/Users/sinaf/OneDrive - NTNU/Work/Pflictarbeit/Teaching/SOS3515_2022/automated_text_analysis"

You can change this to the folder where you analysis materials are stored by using setwd()function like below:

setwd("C:/Users/sinaoz/OneDrive - NTNU/Work/Pflictarbeit/Teaching/SOS3515_2022/automated_text_analysis/graphics")

getwd()

As you can see, my working directory is the graphics folder now. It is generally a good idea to keep all your stuff for an analysis in one folder and set your working directory there. We will explore alternatives in the workflow section.
