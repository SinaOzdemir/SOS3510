A quick introduction to automated text analysis

Sina Oezdemir

2022-08-15

Chapter 1 Introduction:

This book is intended to be a compendium to the lectures of SOS3515 day 3 and 5. Chapters cover the application of the concepts discussed in the lectures. The first four chapters will introduce R as a programming language and get you familiar with working with R. In an ideal world, the course would be semester length and we would spend tutorials of the first several weeks of the class covering these subjects step by step in tutorials. However, our world is far from perfect and this is an intensive course. Therefore, you will need to study the first four chapters by yourself before day 3 lecture as we will use R for automated content analysis.

The book and the course is structured in four parts. First part covers installing and running R on your computer. This compendium will walk you through the basic steps but will not provide extensive instructions on how to install and update R on various operating systems. There are, however, plenty of online resources to help you through.

The second part focuses the basics of R programming. R, unlike STATA and SPSS, is not a software but a programming language, a statistical programming language to be precise. Thus, it enables you to communicate with the computer and provide with instructions with limited inter-mediation. Moreover, it is an open-source programming language like python, so there is a large community of users. This is an advantage in two ways. First of all, it means that if you encounter a problem, chances are other have also encountered the same problem, asked about it online and/or found a solution. Secondly, a good portion of this user community is also developers who like to share their implementations for certain tasks. This means you will rarely have to write your own functions.

The third part will cover the main themes in Natural Language Processing (NLP) also known as automated content analysis (ANA). In this section, the compendium will provide instructions on four different tasks and how to do them in are. These four different tasks are; obtaining textual data, processing textual data for analysis, statistical analysis of textual data, and evaluation of the results. These tasks are the main subjects of day 3 lectures. You should, however, note that NLP is a rapidly developing field. So it is practically impossible to cover all available methods and developments in this compendium. So the compendium is intended to give you the foundation to learn and use any method in the NLP field by yourself.

The last part of the compendium will cover the application of panel data methods. This section is mainly concerned with the data structure of panel data, checks and test for data quality and modelling such data.
